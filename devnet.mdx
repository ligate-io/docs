## Address ladder

Every Ligate identifier is Bech32m-encoded with a typed prefix:

| Prefix    | Identifies                                          |
| --------- | --------------------------------------------------- |
| `lig1...` | account addresses                                   |
| `lpk1...` | public keys (when encoded standalone)               |
| `lsc1...` | schema ids                                          |
| `las1...` | attestor-set ids                                    |
| `lat1...` | attestation ids                                     |
| `ltx1...` | transaction hashes                                  |
| `lph1...` | payload hashes                                      |
| `lcc1...` | NFT collection ids (when `nft` module ships)        |
| `ltk1...` | fungible token denominations (when `tokens` ships)  |

## How users sign in

<Note>
  On `ligate-devnet-1` today: keys live in the local OS keystore via the
  CLI (`ligate keys generate`). A standalone web wallet at
  `wallet.ligate.io` and the Mneme browser extension are Phase 2 (v1).
</Note>

Ligate is a non-EVM Sovereign SDK rollup with ed25519 keys and Bech32m
addresses (`lig1...`). The CLI manages keypairs in the OS keystore at
mode 0600. Pre-extension, integrate via the SDK + CLI.

## Using the faucet

The faucet drips a fixed amount of testnet `$LGT` to any address once
per 24 hours. No auth, no signing, the recipient doesn't need to hold
the private key.

CLI (default endpoint is `api.ligate.io/v1/drip`):

```bash
ligate keys generate --name alice
ligate faucet $(ligate keys show alice)
```

Equivalent HTTP:

```bash
curl -X POST https://api.ligate.io/v1/drip \
  -H 'content-type: application/json' \
  -d "{\"address\":\"$(ligate keys show alice)\"}"
# → {"tx_hash":"ltx1...","amount_nano":"100000000000"}
```

Rate limits are per recipient address. If you're an operator or
node-runner and need more, email
[hello@ligate.io](mailto:hello@ligate.io).

## Service endpoints

### Chain REST (`rpc.ligate.io`)

115 endpoints under `/v1/`, all auto-discovered from the runtime
composition. Point lookups only by design. See [Chain RPC](/rpc) for
the path map and [`docs/protocol/rest-api.md`](https://github.com/ligate-io/ligate-chain/blob/main/docs/protocol/rest-api.md)
in the chain repo for the full reference.

### WebSocket streams

The chain exposes a small set of WebSocket subscriptions under
`/v1/ledger/`. There is no single global `/ws` endpoint. The available
streams:

| Path                                            | Stream                          |
| ----------------------------------------------- | ------------------------------- |
| `/v1/ledger/slots/latest/ws`                    | new slots as they arrive        |
| `/v1/ledger/slots/finalized/ws`                 | slots as they finalize          |
| `/v1/ledger/slots/latest/events/ws`             | events from the head            |
| `/v1/ledger/aggregated-proofs/latest/ws`        | new aggregated proofs           |

Example:

```js
const ws = new WebSocket('wss://rpc.ligate.io/v1/ledger/slots/latest/ws')
ws.onmessage = (e) => console.log(JSON.parse(e.data))
```

### HTTP API (`api.ligate.io`)

Unified service that hosts the faucet drip and the indexer query
endpoints (blocks, txs, addresses, schemas). See the
[ligate-api repo](https://github.com/ligate-io/ligate-api).

## Reset schedule

During devnet, we reserve the right to reset the chain if we ship
breaking protocol changes. Resets are announced on the
[changelog](https://ligate.io/changelog) at least 72 hours in advance.

<Tip>
  Build against semver-tagged spec versions, not the live chain state.
  When we reset, your code keeps working; only historical receipts are
  lost.
</Tip>
