  [github.com/ligate-io/ligate-chain](https://github.com/ligate-io/ligate-chain).
</Info>

The chain's REST API is **point lookups only by design**. List, range,
time-bucketed, by-submitter, aggregation, top-N, and search queries
live in the indexer service, not on the chain.

| Prefix              | Count | Purpose                                                |
| ------------------- | ----- | ------------------------------------------------------ |
| `/v1/ledger/...`    | 17    | Block, batch, transaction, event queries               |
| `/v1/sequencer/...` | 6     | Transaction submission, sequencer status, mempool      |
| `/v1/rollup/...`    | 7     | Chain identity, sync, gas price, schema, simulate      |
| `/v1/modules/...`   | 85    | Per-module state and custom queries                    |

Full reference and per-endpoint detail lands in the next migration pass.
For now, the canonical reference is
[`docs/protocol/rest-api.md`](https://github.com/ligate-io/ligate-chain/blob/main/docs/protocol/rest-api.md)
in the chain repo.
