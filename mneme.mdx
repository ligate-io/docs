Mneme is the first-party browser wallet for Ligate Chain.
Non-custodial, ed25519 native, signs every transaction locally. Ships
with a one-click "Attest with Mneme" button injected into ChatGPT,
Claude, and Gemini, so any AI conversation can produce a verifiable
on-chain receipt without leaving the tab.

<Warning>
  **Mneme is v0.5.** The chain primitives (`attestation` module,
  `themisra.proof-of-prompt` schema) are live on `ligate-devnet-1`.
  The browser-extension surface (standalone wallet UI + AI-tab
  injection) is in development. Until it ships, sign-in lives inside
  the product apps and the [CLI](/cli) covers the same operations.
</Warning>

## What Mneme is (target architecture)

Two surfaces ship together:

| Surface                     | What it does                                                                                  |
| --------------------------- | --------------------------------------------------------------------------------------------- |
| **Browser extension**       | Holds the `lig1...` key, signs `bank.transfer`, `RegisterSchema`, `SubmitAttestation` calls   |
| **AI-tab content script**   | Injects "Attest with Mneme" into ChatGPT, Claude, Gemini. One click hashes the conversation and submits a `themisra.proof-of-prompt` attestation |
| **dApp connector**          | EIP-1193-style provider API. Themisra, Iris, and partner web apps call `mneme.request(...)` for sign + submit |

## Security model

- **ed25519 keypair**, Bech32m-encoded addresses (`lig1...`). Same
  primitive the chain uses natively.
- **BIP-39 seed phrase** for recovery. Standard 12 or 24 words.
- **Fully self-custodial.** Keys live in the extension's encrypted
  storage; the user controls them. Same security model as Phantom or
  MetaMask.
- **MetaMask compatibility** lands at v1 alongside the chain's EVM
  auth lane. Mneme is the canonical wallet today and stays
  first-class after.

## Today, without the extension

Until Mneme ships, the same operations work via the
[CLI](/cli) and the [JavaScript SDK](/js):

```bash
# Generate a keypair (OS keystore, mode 0600)
ligate keys generate --name alice

# Sign and submit a $LGT transfer
ligate transfer --to lig1... --amount 0.5 --signer alice

# Build, sign, submit a themisra.proof-of-prompt attestation
ligate attest \
  --schema themisra.proof-of-prompt/v1 \
  --model-id "anthropic/claude-opus:4.7" \
  --prompt "..." --response ./response.md \
  --signer alice
```

```ts
// Same flow from a Node service or browser app
import { generateKeypair, submitTransfer, submitAttestation } from '@ligate-labs/sdk'
```

## Mneme dApp connector (planned)

When the extension ships, dApps integrate with one snippet:

```ts
// EIP-1193-style provider, namespaced under `window.mneme`.
if (window.mneme) {
  const accounts = await window.mneme.request({ method: 'requestAccounts' })
  const tx = await window.mneme.request({
    method: 'submitAttestation',
    params: { schemaId, payloadHash, signatures },
  })
}
```

Themisra, Iris, and partner schemas integrate the same way.
