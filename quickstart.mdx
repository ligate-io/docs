Install the Ligate CLI, claim testnet $LGT from the faucet, and ship your
first Proof of Prompt attestation. End-to-end in under five minutes.

<Tip>
  **`ligate-devnet-1` is live.** The CLI, faucet, RPC, and explorer are all up
  against Celestia Mocha-testnet. The commands below work; expect breaking
  changes to land in the [changelog](https://ligate.io/changelog) as v1 work
  continues on top.
</Tip>

<Steps>
  <Step title="Install the CLI">
    One-line install, planned for macOS, Linux, and Windows (WSL).

    ```bash
    curl -s https://ligate.io/install | sh
    ligate --version
    # ligate-cli 0.x.x-devnet
    ```

  </Step>

  <Step title="Sign in">
    Email or social login. Keys are generated and custodied for you, recovery
    works through your login. No browser-extension wallet required for v0.5.
    Ligate is a non-EVM Sovereign SDK rollup with ed25519 keys and Bech32
    addresses (`lig1...`).

    ```bash
    ligate auth login --email you@yours.com
    # ✓ Check your email for a sign-in link
    # ✓ Signed in as lig1qxk7r4z9c8m3v2pa6h0w5y8t4d2n3l1f7g6e9b
    ```

  </Step>

  <Step title="Claim testnet $LGT">
    Every new account gets 1000 $LGT from the faucet per day. You need gas to
    submit an attestation transaction.

    ```bash
    ligate faucet claim
    # ✓ 1000 $LGT sent
    #   tx       ltx1zd2qmly…ljgth4
    #   block    1,481,203
    #   explorer https://explorer.ligate.io/tx/ltx1zd2qmly…ljgth4
    ```

  </Step>

  <Step title="Your first attestation">
    The basic shape of a Proof of Prompt: hand the CLI a prompt, a model, and
    (optionally) a response file. It hashes everything, submits the receipt to
    the chain, waits for the attestor quorum, and returns a verifiable receipt
    id.

    ```bash
    ligate attest \
      --model claude-4.7-opus \
      --prompt "Write a recursive descent parser in Rust" \
      --output ./response.md

    # ✓ attestation lat1c3a87e…fd9a7c anchored (celestia-mocha)
    #   quorum 42/62 attestors signed
    #   gas    0.0009 $LGT
    #   block  1,481,207
    ```

  </Step>
</Steps>

## Verify it

Anyone can verify a receipt. Verification checks three things: that the blob
is anchored on Celestia, that the attestor quorum met threshold, and that the
BLS aggregate signature is valid.

```bash
ligate verify lat1c3a87e…fd9a7c
# ✓ receipt anchored (celestia-mocha block 1,481,207)
# ✓ attestor quorum met (42/62 signed, threshold 42/62)
# ✓ BLS aggregate valid
```

<Note>
  The receipt's `prompt_hash` and `output_hash` are SHA-256 over canonical UTF-8
  bytes. See the [Proof of Prompt spec](/proof-of-prompt) for the exact
  canonicalisation rules.
</Note>
