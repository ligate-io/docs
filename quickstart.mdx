Install the Ligate CLI, claim testnet `$LGT` from the faucet, and ship
your first attestation against `ligate-devnet-1`. End-to-end in under
five minutes.

<Tip>
  **`ligate-devnet-1` is live.** The CLI, faucet, RPC, and explorer
  are all up against Celestia Mocha-testnet. The commands below work,
  expect breaking changes to land in the
  [changelog](https://ligate.io/changelog) as v1 hardening continues.
</Tip>

<Steps>
  <Step title="Install the CLI">
    Pre-built tarball install. macOS and Linux supported.

    ```bash
    # Pick PLATFORM = linux-amd64 | linux-arm64 | darwin-arm64 | darwin-amd64
    curl -L -o ligate.tar.gz \
      https://github.com/ligate-io/ligate-cli/releases/latest/download/ligate-PLATFORM.tar.gz
    tar -xzf ligate.tar.gz
    sudo mv ligate /usr/local/bin/
    ligate --version
    ```

    Or build from source: `cargo install --git https://github.com/ligate-io/ligate-cli`.
    See [the CLI page](/cli) for both paths in detail.
  </Step>

  <Step title="Generate a key">
    The CLI manages ed25519 keypairs in the OS keystore at mode 0600.
    Ligate is a non-EVM Sovereign SDK rollup with Bech32m addresses
    (`lig1...`).

    ```bash
    ligate keys generate --name alice
    ligate keys show alice
    # lig1qxk7r4z9c8m3v2pa6h0w5y8t4d2n3l1f7g6e9b
    ```
  </Step>

  <Step title="Claim testnet $LGT">
    The public devnet faucet drips 100 `$LGT` to any address once per
    24 hours.

    ```bash
    # The CLI's `ligate faucet` is currently broken (default URL dead).
    # Use the HTTP endpoint until the next CLI release fixes the default:
    curl -X POST https://api.ligate.io/v1/drip \
      -H 'content-type: application/json' \
      -d "{\"address\":\"$(ligate keys show alice)\"}"
    # ✓ 100 $LGT sent
    #   tx ltx1zd2qmly…ljgth4
    ```

    No signing, no auth, recipient doesn't need to hold the private
    key. See [Devnet, faucet](/devnet#using-the-faucet) for the HTTP
    shape.
  </Step>

  <Step title="Check the balance arrived">
    `--token-id` is the bech32m gas-token id. Pass it once via env so
    you don't have to repeat it.

    ```bash
    export LIGATE_LGT_TOKEN_ID=token_1nyl0e0yweragfsatygt24zmd8jrr2vqtvdfptzjhxkguz2xxx3vs0y07u7

    ligate balance $(ligate keys show alice) --json
    # {
    #   "address": "lig1...",
    #   "token_id": "token_1nyl...",
    #   "amount_nano": 100000000000,
    #   "amount_lgt": 100.0
    # }
    ```

    Amount is in nano `$LGT` (9 decimals). 100,000,000,000 nano =
    100 `$LGT`.
  </Step>

  <Step title="Send your first transfer">
    The simplest write path: a `bank.transfer`. Generate a second
    keypair, transfer `0.5 $LGT` to it.

    ```bash
    ligate keys generate --name bob
    BOB=$(ligate keys show bob)

    # Pull chain identity once.
    export LIGATE_CHAIN_ID=4242
    export LIGATE_CHAIN_HASH=$(ligate info --json | jq -r .chain_hash)

    ligate transfer --to $BOB --amount 0.5 --signer alice
    # ✓ transfer submitted
    #   tx ltx1...
    #   amount 0.5 $LGT to lig1...
    ```

    Verify with `ligate balance $BOB`.
  </Step>
</Steps>

## Submit your first attestation

Attestation is a multi-party flow: register an AttestorSet, register
a Schema, sign with each attestor's key, submit the threshold-signed
attestation. There's no `ligate attest` convenience, the chain
primitives are deliberately decomposed.

For a single-machine smoke test (1-of-1 attestor set with your own
key):

```bash
# 1. Register a 1-of-1 attestor set using alice's own pubkey
ALICE_PK=$(ligate keys show alice --pubkey)
ligate register-attestor-set \
  --members $ALICE_PK \
  --threshold 1 \
  --signer alice
# → returns las1...

# 2. Build your schema definition JSON, then register
cat > schema.json <<EOF
{
  "name": "alice.smoke-test",
  "version": 1,
  "attestor_set_id": "las1...",
  "fee_routing_bps": 0,
  "payload_spec_hash": "<32-byte hex sha256 of a payload JSON-Schema>"
}
EOF
ligate register-schema --file schema.json --signer alice
# → returns lsc1...

# 3. Sign a payload (alice as the only attestor)
echo '{"hello":"world"}' > payload.bin
ligate sign-attestation \
  --schema lsc1... \
  --payload-file payload.bin \
  --submitter $(ligate keys show alice) \
  --signer alice \
  > signatures.json

# 4. Submit
PAYLOAD_HASH=$(sha256sum payload.bin | awk '{print $1}')
ligate submit-attestation \
  --schema lsc1... \
  --payload-hash lph1...$(...convert hex to bech32m...) \
  --signatures signatures.json \
  --signer alice
# → returns lat1...
```

For real Themisra attestations against the canonical federated
quorum, see [Themisra integration](/themisra/api).

## Verify it

Anyone can verify a receipt by querying the chain. The on-chain
record is keyed by `(schema_id, payload_hash)`:

```bash
ligate query attestation <schemaId>:<payloadHash>
# returns the Attestation: submitter, timestamp, signatures
```

Full verification (recomputing `payload_hash` from your local copy of
the off-chain payload, re-checking each attestor signature against the
schema's `attestor_set`) is documented in the
[Proof of Prompt spec](/proof-of-prompt#verification).
